## Issues Identified

### 1. **Insufficient Resources - High Priority**

**Problem:** Current 1GB/1CPU is below Metabase's minimum recommended 3GB/2CPU for any production use.

**Impact:**
- Slow API response times (>200ms)
- Potential OOM kills during async operations
- Poor user experience with concurrent queries

### 2. **Missing Health Check for s60 Host**

**Problem:** No monitoring if s60 host has resource contention from other services.

**Impact:** Metabase may be throttled by host without visibility.

### 3. **Async Processes Not Scheduled**

**Problem:** sync, scan, fingerprinting run automatically without off-peak scheduling.

**Impact:** Background tasks may compete with user queries for resources.

### 4. **Missing Prometheus Metrics Export**

**Problem:** `/api/prometheus` endpoint not enabled/configured.

**Impact:** No observability into Metabase performance.

### 5. **Connection Pool Not Optimized**

**Problem:** Default 15 connections may be insufficient.

**Impact:** Dashboard with 20+ cards may timeout.

## Recommended Actions

### Priority 1: Increase Resources

```yaml
# docker-compose.yml updates
metabase:
  deploy:
    resources:
      limits:
        memory: 3G      # Up from 1G
        cpus: "2.0"     # Up from 1.0

metabase-db:
  deploy:
    resources:
      limits:
        memory: 2G      # Up from 512M
        cpus: "1.0"     # Up from 0.5
```

**Estimated Impact:** +$10-20/month if cloud, otherwise free on bare metal s60.

### Priority 2: Add Connection Pool Setting

```yaml
environment:
  MB_APPLICATION_DB_MAX_CONNECTION_POOL_SIZE: 25
```

### Priority 3: Enable Prometheus Metrics

```yaml
environment:
  MB_PROMETHEUS_ENABLED: "true"
```

### Priority 4: Configure Async Scheduling

Metabase doesn't have built-in scheduling UI - need to set sync hours via:
```yaml
environment:
  MB_ANALYTICS_ENABLED: "false"  # Reduces background processing
```

### Priority 5: Add Host Monitoring

Create script to monitor Docker resource usage:
```bash
docker stats --no-stream --format "table {{.Name}}\t{{.CPUPerc}}\t{{.MemUsage}}"
```
